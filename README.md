# 日志搜索系统

一个基于Web的分布式日志搜索工具，支持跨服务器关键字搜索，实时显示搜索结果。

## 功能特性

### 服务器管理
- 添加/编辑/删除系统和服务器
- 支持SSH连接远程服务器
- 测试服务器连接状态
- 复制服务器配置
- 批量修改文件匹配规则

### 日志搜索
- 跨服务器关键字搜索（不区分大小写）
- 实时流式显示搜索结果
- 支持配置上下文行数（0-50行）
- 文件名模糊匹配（支持多个模式，逗号分隔）
- 搜索结果倒序显示（最新日志在前）
- 支持带空格的关键字（无需加引号）
- 关键字长度限制200字符

### 日志下载
- 支持下载搜索结果为ZIP压缩包
- 按服务器名称分目录存储

### 日志记录
- 详细的操作日志记录
- 按天自动分片存储（保留30天）
- 记录所有API调用和SSH命令

## 环境要求

- Python 3.7+
- pip3

## 快速开始

### 1. 依赖检查与安装

```bash
cd /opt/log_search
./libs/install.sh
```

该脚本会自动检查：
- Python版本是否 >= 3.7
- pip3是否已安装
- Flask和Paramiko是否已安装

如果依赖已满足，脚本会跳过安装；否则会从离线包安装依赖。

### 2. 启动应用

```bash
./start.sh start
```

启动时会自动检查依赖，如果缺失会自动安装离线依赖包。

### 3. 访问应用

打开浏览器访问：`http://服务器IP:5001`

### 4. 停止应用

```bash
./start.sh stop
```

## 启动脚本命令

### Kylin V10 部署说明

**x86_64 版本：** 直接使用离线包安装即可。

**ARM 版本：** 需要在有网络的ARM服务器上重新生成离线包：

```bash
# 在有网络的ARM服务器上执行
mkdir libs && cd libs
pip3 download flask paramiko -d .
```

然后将 `libs/` 目录复制到目标服务器。

## 使用说明

### 添加系统和服务器

1. 点击"添加系统"按钮，输入系统名称
2. 在系统下点击"添加服务器"
3. 填写服务器信息：
   - IP地址：服务器IP
   - 端口：SSH端口（默认22）
   - 用户名：SSH登录用户
   - 密码：SSH登录密码
   - 日志目录：日志文件所在目录
   - 文件名匹配：可选，逗号分隔的文件名关键字

### 搜索日志

1. 点击系统右侧的"搜索日志"按钮（在新标签页打开）
2. 输入搜索关键字
3. 设置上下文行数（0-50，默认0）
4. 点击"搜索"按钮
5. 左侧显示服务器和文件树，点击文件查看日志内容

### 文件名匹配规则

- 留空：匹配目录下所有文件
- 单个关键字：`app` → 匹配文件名包含"app"的文件
- 多个关键字：`app,error,sys` → 匹配包含任一关键字的文件

### 批量修改文件匹配

点击"批量修改文件匹配"按钮，可一次性修改该系统下所有服务器的文件匹配规则。

## API接口

### 系统管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/systems | 获取所有系统 |
| POST | /api/systems | 添加系统 |
| PUT | /api/systems/{name} | 更新系统 |
| DELETE | /api/systems/{name} | 删除系统 |

### 服务器管理

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/systems/{name}/servers | 添加服务器 |
| PUT | /api/systems/{name}/servers/{index} | 更新服务器 |
| DELETE | /api/systems/{name}/servers/{index} | 删除服务器 |
| POST | /api/systems/{name}/servers/{index}/test | 测试连接 |

### 日志搜索

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/search | 搜索日志（一次性返回） |
| POST | /api/search/stream | 搜索日志（流式返回） |
| POST | /api/search/download | 下载搜索结果（ZIP） |

## 日志说明

### 日志位置

```
/opt/log_search/logs/app.log           # 当天日志
/opt/log_search/logs/app.log.2026-07-05  # 历史日志
```

### 日志内容

记录所有操作，包括：
- 查询系统列表
- 添加/删除/更新系统和服务器
- 测试连接
- 搜索日志（含执行的SSH命令）
- 下载日志

### 日志格式

```
[2026-07-05 16:30:09] INFO [添加服务器] 成功: 系统="测试系统", 服务器=192.168.1.100:22, 用户=test, 目录=/var/log, 文件匹配=app,error
[2026-07-05 16:30:10] INFO [搜索服务器] 192.168.1.100:22 执行命令: grep -F -n -i -A 0 -B 0 -- 'error' '/var/log/app.log'
```

## 配置说明

### 修改端口

编辑 `app.py` 文件最后一行：

```python
app.run(host='0.0.0.0', port=5001, debug=False)
```

修改 `port=5001` 为其他端口，然后重启应用。

### 修改日志保留天数

编辑 `app.py` 文件中的日志配置：

```python
file_handler = TimedRotatingFileHandler(
    ...
    backupCount=30,  # 修改此值
    ...
)
```

## 常见问题

### 1. 端口被占用

```
OSError: [Errno 98] Address already in use
```

解决方案：修改 `app.py` 中的端口号，或停止占用端口的进程。

### 2. SSH连接失败

检查：
- 服务器IP和端口是否正确
- 用户名和密码是否正确
- 服务器是否开启SSH服务
- 防火墙是否允许SSH连接

### 3. 搜索无结果

检查：
- 关键字是否正确
- 日志目录路径是否正确
- 文件名匹配规则是否正确
