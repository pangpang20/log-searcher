# PRD: 日志搜索系统

## 需求背景

运维团队在日常工作中需要频繁查看和分析分布在多台服务器上的日志文件。目前的操作方式是手动SSH登录到每台服务器，使用grep等命令搜索日志内容，这种方式效率低下且容易遗漏关键信息。特别是当需要跨多个服务器、多个目录进行关键字搜索时，操作繁琐且耗时。

本系统旨在提供一个统一的Web界面，让用户能够集中管理服务器信息，并通过简单的操作在多台服务器的日志文件中进行关键字搜索，大幅提升运维效率。

## 目标与价值

**目标：**
- 提供可视化的服务器管理界面，支持添加、编辑、删除服务器信息
- 支持按系统名称对服务器进行分组管理
- 提供跨服务器的关键字搜索功能，支持不区分大小写
- 搜索结果可配置显示上下文行数（默认前后20行）
- 支持搜索结果的本地下载

**价值：**
- 减少运维人员手动登录服务器的时间
- 提高日志分析效率，快速定位问题
- 集中管理服务器信息，降低管理复杂度
- 支持团队协作，共享服务器配置

## 术语/角色定义

- **系统名称**：用于对服务器进行逻辑分组的标识，如"订单系统"、"支付系统"等
- **服务器**：需要连接的远程主机，包含IP地址、端口、认证信息等
- **目录**：服务器上存放日志文件的路径
- **SSH**：Secure Shell，用于安全连接远程服务器的协议
- **关键字搜索**：在日志文件内容中查找匹配的文本

## 适用范围

- 适用：运维团队、开发团队的日志查看需求
- 适用：通过SSH可访问的Linux服务器
- 适用：文本格式的日志文件
- 不适用：实时日志流监控
- 不适用：日志的修改或删除操作

## 非目标/不包含功能

- 不包含实时日志流监控（tail -f）
- 不包含日志文件的修改或删除功能
- 不包含日志分析和统计功能
- 不包含用户权限管理（单用户系统）
- 不包含日志告警功能

## 功能需求

**FR-1: 系统管理功能**
- 用户可以添加新的系统，包含系统名称
- 系统名称必须唯一，不能重复
- 用户可以删除系统（同时删除关联的服务器信息）
- 用户可以编辑系统名称

**FR-2: 服务器管理功能**
- 用户可以在系统下添加服务器，包含：IP地址、端口（默认22）、用户名、密码、目录路径
- 一个系统下可以添加多台服务器
- 用户可以编辑服务器信息
- 用户可以删除服务器
- 系统自动验证服务器连接信息的有效性

**FR-3: 数据持久化功能**
- 所有数据保存到本地JSON文件中
- 数据文件存储在应用目录下的data文件夹
- 支持数据的读取和写入
- 应用启动时自动加载历史数据

**FR-4: 系统列表展示功能**
- 首页展示所有已添加的系统列表
- 点击系统名称展开显示该系统下的所有服务器
- 显示每台服务器的IP、端口、目录信息
- 支持系统和服务器的编辑、删除操作

**FR-5: 日志搜索功能**
- 每个系统列表项提供"搜索日志"按钮
- 点击后弹出搜索对话框
- 用户输入搜索关键字
- 系统自动连接该系统下所有服务器的对应目录
- 在所有日志文件中进行不区分大小写的关键字搜索
- 支持配置显示上下文行数（默认前后20行，可修改）

**FR-6: 搜索结果展示功能**
- 按服务器分组展示搜索结果
- 每个服务器下显示匹配的日志文件
- 日志文件下显示匹配的行内容（包含上下文）
- 未找到匹配内容的服务器显示"未找到结果"提示
- 结果按服务器名称、文件名排序

**FR-7: 日志下载功能**
- 搜索结果页面提供"下载日志"按钮
- 将当前展示的搜索结果导出为文本文件
- 文件内容包含服务器名称、文件路径、匹配内容
- 自动下载到用户本地

## 关键流程/交互描述

**流程1：添加服务器**
1. 用户点击"添加系统"按钮
2. 输入系统名称，点击确认
3. 系统创建成功，显示在列表中
4. 用户点击"添加服务器"
5. 输入IP地址、端口、用户名、密码、目录路径
6. 系统验证连接信息
7. 保存服务器信息

**流程2：搜索日志**
1. 用户在系统列表中点击"搜索日志"
2. 弹出搜索对话框
3. 输入搜索关键字
4. 可选：修改上下文行数（默认前后20行）
5. 点击"搜索"按钮
6. 系统连接所有服务器并搜索日志
7. 搜索结果实时显示在界面上
8. 用户可点击"下载"保存结果

**流程3：搜索结果展示顺序**
1. 先显示服务器名称（如：192.168.1.100）
2. 再显示日志文件路径（如：/var/log/app.log）
3. 最后显示匹配的日志内容（包含上下文）
4. 多个匹配点之间用分隔线区分

## 风险与依赖

**风险：**
- 网络连接不稳定可能导致SSH连接中断
- 大日志文件搜索可能较慢（100MB-1GB）
- 服务器密码明文存储存在安全风险

**依赖：**
- 服务器需要开启SSH服务
- 需要Python 3.x运行环境
- 需要安装paramiko库用于SSH连接
- 需要安装Flask框架

**缓解措施：**
- SSH连接设置超时机制
- 大文件采用流式读取，避免内存溢出
- 后续版本可考虑加密存储密码
